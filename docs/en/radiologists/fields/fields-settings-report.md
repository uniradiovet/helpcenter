# Report settings fields

Understand what each field in the report settings (PDF and exam) means.

---

## Report settings (edit_report)

Controls the appearance and content of the report PDF generated by the system.

---

### **Logo**

**PDF header logo**: Image displayed in the report PDF header. Accepted formats: JPEG or PNG.

---

### **Email signature**

**Email signature**: Text displayed in the signature of notification emails sent by the system (e.g., service name, contact, website).

---

### **Dimensions**

**Logo height (in px)**: Defines the height in pixels of the radiologist's logo displayed in the PDF. Example: `50`.

**Client logo height (in px)**: Defines the height in pixels of the client clinic's logo displayed in the PDF, when applicable. Example: `50`.

**Font size (in px)**: Defines the font size of the report text in the PDF. Example: `13`.

---

### **Report texts**

**Report title**: Text displayed as the title in the report PDF (e.g., "Veterinary Imaging Report").

**End-of-report note**: Footer text displayed at the end of the report content in the PDF (e.g., terms, general observations).

**End-of-attachments-page note (images)**: Footer text displayed at the end of the attached images page in the PDF.

---

### **PDF layout**

**Images per page**: Defines how many images are displayed per page in the images section of the PDF (options: 1, 2, 4, 6, 8, or 10).

---

### **Content displayed in the report**

Controls which sections of the patient's clinical state are displayed in the report PDF:

| Field | Description |
|---|---|
| **Show clinical history** | Displays the patient's clinical history in the PDF. |
| **Show clinical signs** | Displays clinical signs in the PDF. |
| **Show clinical suspicion** | Displays clinical suspicion in the PDF. |
| **Show QR Code** | Displays a QR Code in the PDF with the link to access the report online. |

---

### **Public report sharing**

Defines what is displayed when the owner or clinic accesses the public report link:

| Field | Description | Default |
|---|---|---|
| **Share attached images** | Displays exam images in the public view. | Enabled |
| **Share attached files** | Displays non-image files (PDFs, documents, etc.) in the public view. | Enabled |
| **Share external link** | Displays external links (PACS, Dropbox, Google Drive, etc.) in the public view. | Enabled |

::: tip TIP
The sharing settings defined here serve as global defaults. Each clinic can individually override these values in their own account settings.
:::

---

## Exam settings (edit_exam)

Controls the behavior of the exam creation form used by clinics.

---

### **Exam types**

**Exam types**: List of exam modalities available for selection in the form. Each type can be:

- **Enabled/Disabled**: Defines whether the type appears as an option for clinics.
- **Color**: Visual identification color of the exam type in the exam list (useful for quick differentiation in the queue).

---

### **Urgent exams**

**Allow clients to mark exams as urgent**: When enabled, the urgency field becomes visible in the exam creation form for clinics.

**Urgency field description**: Explanatory text displayed below the urgency option in the clinic's form. Use it to inform the additional fee, delivery timeframe, or other conditions.

Example: *"Urgent exam fee: $30.00. Timeframe: Up to 3 hours."*

---

### **Customize fields**

Configuration table for visibility and required status of the exam form fields for clinics. For each field it is possible to define:

| Setting | Description |
|---|---|
| **Visible** | The field is displayed in the exam creation form. |
| **Required** | The field must be filled in before submission. Can only be enabled if the field is visible. |

Configurable fields include: patient name, species, breed, weight, age, sex, clinical history, clinical signs, clinical suspicion, and requesting physician.
